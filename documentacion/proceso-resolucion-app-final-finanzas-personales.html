Proceso Resolución - App Final (Gestión Finanzas Personales)

Como trabajo final para la asigantura de Desarrollo de Interfaces, el objetivo ha sido cear una aplicación de Gestión de Finanzas Personales.

Para la resolución de esta tarea se ha utilizado principalmente el entorno de desarrollo Antigravity para todo lo relacionado con el código y el diseño de la aplicación. También se ha usado ChatGPT para mejorar algunos de los prompts que se le ha pasado a Antigravity, para la preparación previa de la app y para solucionar algunos problemas en los códigos.

A lo largo de este proceso de resolución se desarrollarán los siguientes puntos:

1.-Vistazo a la aplicación final

Primero veremos el resultado final de la aplicación, explicando las distintas pantallas y su funcionalidad.

Como se dijo anteriormente, es una aplicación de gestión de finanzas personales, que permitirá al usuario manejar sus gastos e ingresos creando y guardando sus transacciones, navegando fácilmente a través de ellas, pudiendo adjuntar archivos, crear sus propias categorías para el tipo de transacción y observar distintos tipos de estadísticas a través de tablas.

--Pantallas de la App--

-Lo primero que se verá al abrir la app, si no se tiene una cuenta, será la pantalla de Login. El usuario podrá elegir entre registrarse o iniciar sesión si ya tiene una cuenta.

 

 

-Al entrar, lo primero que se verá será la sección Dashboard. Aquí, el usuario podrá ver su balance total, junto a los gastos e ingresos totales, y una lista de las últimas transacciones añadidas. En esta pantalla será donde podrá añadir nuevas transacciones.

Al dar a +Añadir, aparecerá una ventana para rellenar los distintos campos necesarios para crear una nueva transacción.

A las transacciones se le podrán añadir distintos archivos, como imágenes o pdfs. Esto se representará visualemente con un símbolo de un clip, y al seleccionar esa transacción se podrá visualizar el contenido.

 

-En la sección Transacciones se podrán ver todas las que se hayan creado. Hay una barra de búsqueda que servirá para encontra una transacción por su nombre, además de distintos filtros de búsqueda.

 

-En Categorías el usuario podrá crear sus propias categorías tanto para gastos como ingresos, para así personalizarlas a su gusto.

 

-En Estadísticas se mostrarán distintos gráficos para visualizar distintos tipos de datos relacionados con las transacciones y sus categorías.

 

-Por último, en la sección Perfil el usuario podrá modificar el aspecto de la aplicación, eligiendo entre modo claro y oscuro y el tema principal. También hay una opción para el borrado de datos de la cuenta.

4.-Generar un APK

Otro de los objetivos de esta práctica era generar un APK de la aplicación para poder ser usada en móviles. A continuación se mostrará el proceso seguido para su creación y capturas de pantalla de esta versión.

Genrando el APK

-Desde el directorio del proyecto, se inicia un proyecto npm

-Se instala Capacitator, un medio que permitirá crear una aplicación móvil a partir de nuestra app web actual.

-Con el comando npx cap init se inicializa Capacitator, y nos preguntará por un nombre y un ID único para la App. Habrá que crear previamente una carpeta ('www'), en la que estén todos los componentes que forman la app.

-Se crea la carpeta android (nombre del proyecto), que contendrá todo lo necesario para generar un proyecto para Android Studio.

-Con npx cap copy android se sincronizará la aplicación con Android. Este será el comando que habrá que utilizar cada vez que se necesite actualizar la versión de móvil para generar el nuevo APK. Previamente habrá que añadir la nueva versión de los archivos de la aplicación a la carpeta www.

-Se abre el proyecto en Android Studio. Aquí habrá que ir a Build y seleccionar Generate APKs. Se creará el archivo .apk en la carpeta android.

Para usar la aplicación en el móvil habrá que pasar este archivo y descargarlo desde nuestro teléfono.
