Proceso Resolución - Visor Documentos

Para esta tarea el objetivo ha sido una aplicación de visor de documentos. En este documento redactaré los pasos seguidos, prompts usados y las secciones del código que se encargan de los aspectos principales de la aplicación.

 

Introducción a la aplicación

Esta aplicación es capaz por un lado de abrir archivos que se encuentren en el sistema para visualizarlos en la aplicación, además de poder editarlos y descargar estas ediciones.

En esta aplicación también es posible crear archivos desde cero, con opciones típicas de un editor de archivos.

Descarga de archivos desde la app:

La última función que incluye es una Biblioteca de Archivos, en la que se pueden guardar todos los archivos creados o editados en la aplicación. En esta sección se incluyen las opciones 'Abrir', 'Clonar' y 'Eliminar' para los archivos. La primera opción permitirá continuar editando o descargar el archivo.

 

Resolución

Lo primero que tuve que hace fue preparar Firebase. Cree un proyecto y obtení el siguiente fragmento de código que me serviría para vincular el proyecto al Firebase. 

La IA me guió para adaptar el script al código y finalmente se muestra así, excluyendo algunas funciones que se verán más adelante.

Una vez preparado Firebase, le pase un prompt largo a la IA con todas las funcionalidades que tendría que tener la app, resultando en lo mostrado anteriormente. A continuación mostraré el prompt en partes acompañado de los fragmentos de código que se encargan de cumplir con lo pedido.

Lo primero ha sido una pauta de como debe actuar y una breve introducción al proyecto y la pantalla principal. En el código, la función renderHome() se encarga de mostrar las opciones y darles su funcionalidad.

La primera instrucción ha sido la funcionalidad que tendrá la opción 'Abrir Archivo'.

El fragmento window.onLoad se encarga de activar la función handleFileUpload(event) cada vez que se selecciona un archivo, la cual se encarga de validar y abrir los archivos seleccionados.

En el siguiente fragmento del prompt se explican las distintas opciones que deberían aparecer tras abrir un archivo, como la posibilidad de editarlo y descargarlo.

 

Para empezar, la función renderHeader() muestra la barra de opciones superior, y cambia sus elementos dependiendo de si se está visualizando o editando el archivo.

 

La función toggleEditMode() se encarga de intercambiar entre modo visualización y modo edición, y es llamada en la función anterior.

 

Por último, la función handleSave() se encarga de verificar que haya un archivo para descargar y pide un nombre y extensión al usuario. Entonces llamará a la función downloadFile(), que se encarga de la descarga.

Ahora con la segunda opción, 'Crear Archivo'. Primero, la función createNewFile() se encarga de crear un documento. Llama a su vez a la función navigate(), para activar la vista de edición.

 

Las funciones renderEditorToolbar() y applyEditorCommand() se encargan de mostrar las herramientas de edición.

Las mismas funciones handleSave() y downloadFile() se usarán igual en este apartado.

Para la tercera mayor funcionalidad, la Biblioteca de Archivos, el prompt ha sido el siguiente:

Primero, la función saveToLibrary() se encarga de guardar los archivos en Firestore y de actualizarlos.

La función setUpLibraryListener() es la que se encarga de actualizar los archivos y cambios en la Biblioteca, y renderLibrary() muestra el formato de todos los archivos guardados en la Biblioteca junto con los respectivos botones de 'Abrir', 'Clonar' y 'Eliminar'.

Dentro de la función renderApp(), el siguiente fragmento es el que se encarga de activar la funcionalidad de cada botón para los archivos en la Biblioteca. El de 'Abrir' llama a navigate(), el de 'Clonar' llama a la función cloneFile() y el de 'Eliminar' a la función deleteFile().
